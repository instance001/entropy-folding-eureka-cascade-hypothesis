## First-Pass Minimal Glossary

### Entropy folding

Primary source posture:

- a proposed capacity-raising or capacity-shifting process
- unresolved items are vaulted, reorganized, and tracked against later performance changes
- treated as thermodynamically compatible under open-system assumptions in the more careful source set

What it is not:

- not a claim of new physics
- not guaranteed performance gain
- not already established empirical law

Primary sources:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/paper.md`
- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/GLOSSARY.md`

### Vault / vaulted entropy (`E_v`)

Primary source posture:

- logged unresolved tasks, problems, or load retained for later handling
- structured backlog rather than discarded residue

Variation to note:

- some sources emphasize unresolved tasks
- some downstream sources emphasize unresolved cognitive load more generally

What it is not:

- not an energy measurement
- not simple deletion or suppression

Primary sources:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/GLOSSARY.md`
- `Symbound-Entropy-Architecture/GLOSSARY.md`

### Folding (`F`)

Primary source posture:

- explicit reorganization steps applied to vaulted material
- examples include clustering, constraint rewrites, and ordering changes

What it is not:

- not direct solving by itself
- not hidden inference that cannot be operationally described

Primary sources:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/paper.md`
- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/GLOSSARY.md`

### Capacity / capacity proxy (`M`, `ΔM`)

Primary source posture:

- estimated through proxies rather than claimed as a directly measured inner quantity
- common proxies include time-to-solve, error rate, plan compression, backtracking reduction, and transfer performance

What it is not:

- not ground-truth capacity
- not a stable scalar guaranteed to map cleanly across all tasks or substrates

Primary sources:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/README.md`
- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/GLOSSARY.md`

### Insight (`I`)

Primary source posture:

- discrete performance jump or abrupt downstream resolution following folding

What it is not:

- not guaranteed after every fold
- not cost-free
- not proof of the theory by itself

Primary sources:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/paper.md`
- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/GLOSSARY.md`

### Project / output (`P`)

Primary source posture:

- concrete outputs or artifacts created after an insight or resolution event
- these outputs feed the next cycle by generating new unresolved items

What it is not:

- not validation merely because an output exists
- not evidence that the preceding fold was correctly characterized

Primary source:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/paper.md`

### Scope

Primary source posture:

- the declared domain in which entropy-folding claims are meant to apply
- includes boundary conditions, escalation paths, and termination criteria

Why it matters:

- the source corpus explicitly warns that scope mistakes can masquerade as entropy relief

What it is not:

- not an optional appendix
- not something to be left implicit

Primary source:

- `entropy-folding-foundational-frameworks/entropy-folding-foundation-scope/paper.md`

### Scale

Primary source posture:

- scale is framed, in the foundational-scale material, as a function of entropy, stability, and resolution rather than only size or magnitude

Why it matters here:

- it helps explain why some downstream materials attempt to connect local fold dynamics to broader regime changes

Current caution:

- this is important background structure, but it is not necessary to over-import the entire scale framework into the minimal core hypothesis

Primary source:

- `entropy_folding_scale/README.md`

### Vector field / state space / trajectory / attractor / boundary

Primary source posture:

- these are the main structural terms in the vector-theory layer
- systems are treated as occupying states in an abstract field
- trajectories follow lawful directional pressures
- attractors are regions trajectories converge toward
- boundaries are regions where trajectories are constrained, redirected, or undefined

Why they matter here:

- they provide a substrate-agnostic scaffold for discussing directional change and folding without restricting the model to one domain

Current caution:

- this layer is formal and theoretical
- it should clarify structure, not be mistaken for empirical confirmation

Primary sources:

- `Entropy-Folding-Vector-Theory/paper/Paper_v1.0.md`
- `Entropy-Folding-Vector-Theory/GLOSSARY.md`

### TAEP cycle

Primary source posture:

- temporal asymmetric entropy processing
- storing unresolved complexity now and resolving later when capacity increases

Why it matters:

- this is one of the clearer downstream names for deferred-resolution logic

Current caution:

- useful operational vocabulary, but more downstream than the minimal upstream hypothesis statement

Primary source:

- `Symbound-Entropy-Architecture/GLOSSARY.md`

### Failure metabolism

Primary source posture:

- a downstream learning function in which mistakes are converted into later structural gains

Current caution:

- important for reading downstream architecture documents
- should remain secondary to the narrower fold-capacity-insight cycle in the core hypothesis repo

Primary source:

- `Symbound-Entropy-Architecture/GLOSSARY.md`
