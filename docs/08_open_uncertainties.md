# Open Uncertainties

## Purpose

Preserve unresolved ambiguity instead of smoothing it away during consolidation.

## Material That Belongs Here

- terminology drift
- unresolved relation between entropy folding and eureka cascade if not explicit
- inconsistent scope language across repos
- speculative passages that should remain labeled speculative

## Material That Must Not Go Here

- creative reconciliation
- hidden synthesis
- argument by confidence or tone

## Drafting Rule

If uncertainty remains after reading source material, record it plainly.

## First-Pass Open Uncertainties

These are uncertainties that remain visible in the source corpus after the first consolidation pass.

They are recorded here to prevent false neatness.

## 1. Exact Relation Between "Entropy Folding" And "Eureka Cascade"

Current status:

- entropy folding is the clearer and more basic term in the core source set
- eureka-cascade language appears to function more as a downstream descriptive label for clustered or repeated insight phenomena

What remains uncertain:

- whether the corpus ever defines a strict formal equivalence
- whether eureka cascade is a subset, consequence, or rhetorical alias of the entropy-folding cycle

Current handling in this repo:

- treat entropy folding as primary
- treat eureka cascade as downstream unless a tighter relation is explicitly sourced

## 2. How Much Weight To Give Downstream Integrative Repos

Current status:

- some downstream repos contain helpful clarifications, limitations, and operational language
- those same repos also sometimes use broader, stronger, or more sweeping rhetoric

What remains uncertain:

- how much of that broader rhetoric should be read as mature formalization versus exploratory extension

Current handling in this repo:

- downstream sources may clarify
- they do not automatically outrank the narrower core sources

## 3. Status Of Thermodynamic Language Across The Corpus

Current status:

- the more careful sources explicitly reframe thermodynamic language as open-system compatibility
- some broader downstream texts push closer to stronger, more law-like wording

What remains uncertain:

- whether the corpus as a whole is fully stabilized around the narrower formulation

Current handling in this repo:

- prefer the open-system compatibility reading unless a stronger claim is explicitly justified

## 4. Capacity As Proxy Versus Capacity As Real Quantity

Current status:

- the disciplined core source repeatedly treats capacity change as proxy-tracked
- some downstream materials speak more naturally as if the quantity were directly real and operational

What remains uncertain:

- whether the theory intends a future direct measure or whether proxy status is a permanent methodological constraint

Current handling in this repo:

- treat capacity as proxy-governed in the core statement

## 5. Level Of Empirical Maturity

Current status:

- the source corpus contains falsifiability language, null models, replication talk, and methodological proposals
- this does not yet establish that the relevant empirical program has already been carried out at convincing scale

What remains uncertain:

- how much of the work is conceptual scaffold versus demonstrated result

Current handling in this repo:

- present the framework as an operationalizable research program, not an already-validated science

## 6. Scope Breadth Across Domains

Current status:

- multiple sources present the framework as substrate-agnostic or widely generalizable
- the foundational-scope material simultaneously warns that claims are meaningful only within declared domain boundaries

What remains uncertain:

- how broad a declared domain can be before the framework loses explanatory sharpness

Current handling in this repo:

- preserve the tension instead of resolving it

## 7. Cross-Substrate Generalization

Current status:

- several tools and downstream documents imply or propose relevance across human, AI, and hybrid systems
- methodological language for cross-substrate validation exists in downstream materials

What remains uncertain:

- whether the corpus currently demonstrates enough to treat that generalization as strong rather than aspirational

Current handling in this repo:

- note cross-substrate ambition
- do not treat ambition as established equivalence

## 8. Signal Versus Theory Boundary

Current status:

- some sources sit awkwardly between hypothesis articulation and cross-domain resonance
- this is especially true for materials that are partly theoretical, partly analogical, and partly methodological

What remains uncertain:

- whether some sources should ultimately live more naturally in repo one or repo two

Current handling in this repo:

- keep the repo boundary conservative for now
- prefer moving ambiguous sources into the atlas or appendix rather than over-centralizing them in the hypothesis spine

## 9. Tool Output Status

Current status:

- some tools have example outputs present
- some only describe intended outputs or methods

What remains uncertain:

- which existing outputs are illustrative scaffolds versus outputs that were used substantively in the wider source program

Current handling in this repo:

- label tool-derived outputs clearly
- do not silently upgrade them into stronger evidence

## 10. Vocabulary Stability

Current status:

- the corpus contains recurring terms, but not always with perfectly stable emphasis across repositories

Examples:

- vault
- entropy
- capacity
- scale
- attractor
- intuition

What remains uncertain:

- how much drift reflects genuine theoretical growth versus loose reuse of a shared vocabulary

Current handling in this repo:

- document term variation where it matters
- avoid forcing a false single-definition regime
