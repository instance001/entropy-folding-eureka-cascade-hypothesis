# Reader Guide

## Purpose

This document explains how to read this repository.

## Intended Use

Treat this repo as a clarified map of existing source material, not as a final settled theory.

## Reading Posture

- Read claims in proportion to their source support.
- Distinguish framework language from empirical support.
- Distinguish conceptual architecture from downstream analogy or application material.
- Treat unresolved ambiguity as unresolved.

- orientation for outside readers

- how the repo separates strong, weak, and uncertain material

## Epistemic Posture

The strongest current posture in the source corpus is narrow and disciplined:

- entropy folding is presented as a proposed mechanism
- claims should be operationalized with proxies rather than treated as settled ground truth
- thermodynamic language is framed as open-system compatibility, not new physics
- results should be compared against null models
- uncertainty, abstention, and scope declaration matter

## How To Treat Downstream Material

Some downstream repositories are useful because they:

- restate the framework in plainer language
- expand operational implications
- name limitations, boundary conditions, or methods

But downstream material should not automatically be treated as stronger than the narrower upstream framing.

In practice, that means:

- use downstream summaries to clarify, not to escalate
- preserve stronger source claims only where they are explicit
- do not let expansive wording erase earlier cautions

## Reader Warning

Several source repos use ambitious vocabulary. This consolidation favors the narrower reading when there is tension between:

- formal caution and rhetorical breadth
- proposed mechanism and asserted law
- operational proxy and claimed ground truth

Where ambiguity remains, it is recorded rather than resolved.
