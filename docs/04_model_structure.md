## Layer 2: Operational Fold Cycle

This is the clearest central layer for the hypothesis repo.

Core sequence:

1. unresolved items are retained as vaulted material
2. stall or unresolved complexity is recognized
3. explicit folding operations reorganize that material
4. capacity change is estimated with proxies
5. later insight or performance shifts are observed
6. concrete outputs feed the next cycle

Primary source:

- `entropy-folding-as-directed-thermodynamics-for-cognition-finished/paper.md`

Role in the hypothesis:

- provides the most concrete account of what the model says happens
- provides the cleanest operational bridge between abstract theory and measurable practice

## Operational Mechanism Detail

The working structural mechanism for this repo can be stated more explicitly as:

1. a system meets unresolved complexity it cannot presently metabolize
2. that material is vaulted instead of fully resolved
3. the system diverts, routes around, or pivots away
4. later, a changed angle, scale, context, topology, or catalyst recontacts the vaulted material
5. if the fit is good enough, the vault metabolizes rapidly
6. the first unlock may expose adjacent vaulted material, creating a linked or cascading sequence

This adds an important distinction:

- the fold cycle is not only "problem -> reorganization -> insight"
- it is more specifically "stuckness -> vaulting -> later fit -> unlock -> possible adjacency cascade"

That distinction matters because it is the mechanism being used to judge atlas entries.

## Structural Relationship Map

The safest first-pass relationship map is:

- foundational constraints define where claims are allowed to apply
- the fold cycle describes the proposed mechanism within that bounded space
- downstream vocabulary provides richer operational labels for recurring parts of that cycle
- downstream integrations attempt to connect the cycle to wider systems, scales, or architectures

More concretely, the fold cycle should be read here as:

- unresolved tension is not erased
- it remains latent or deferred
- later structural fit can rapidly change its metabolizability
- one unlock may expose neighboring unresolved structures

## Why Vector And Scale Material Matter Here

The vector-theory layer matters because it supplies a substrate-agnostic way to talk about:

- state
- directionality
- pressure
- attractors
- boundaries
- convergence

The scale material matters because it suggests that:

- system behavior may change qualitatively across scales
- local and larger-scale behavior should not be assumed to be identical

These two layers help explain why later repos talk about:

- burst clusters
- phase shifts
- cross-domain correspondences

But in this repo they should be used as framing support, not as an excuse to import every downstream extrapolation.

## Model-Structure Caution

The source corpus is not perfectly uniform. Some documents are highly disciplined and operational, while others are broader and more programmatic.

This repo therefore treats the model structure as:

- core operational cycle at the center
- bounded by scope and theoretical constraints
- surrounded by downstream elaborations that may clarify parts of the structure but do not automatically raise confidence in the core hypothesis
